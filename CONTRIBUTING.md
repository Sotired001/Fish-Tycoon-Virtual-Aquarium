# Contributing to Fish Tycoon: Virtual Aquarium

Thanks for your interest — contributions are welcome!
This project is an early prototype, so the best way to contribute right now is to open issues that describe your idea, add design docs to `docs/`, or submit small PRs with focused changes.

## Getting started
1. Fork the repository and create a new branch.
2. Run the project locally (`npm install` then `npm run dev`).
3. Open an issue explaining the change you want to make.
4. If you make changes, submit a PR referencing the issue.

## Where to begin
- `constants.ts` — Add or tweak fish species and upgrade values.
- `services/store.ts` — Modify the game rules and persistence behavior.
- `components/` — UI and canvas interaction behavior.

## Design doc format (place docs in `docs/`)
Use a descriptive title, state, owner, and acceptance criteria.

```
# Title
Status: Draft | Reviewing | Accepted
Owner: @yourhandle
Last Updated: YYYY-MM-DD

## Overview
## Goals
## Non-Goals
## Specification
## Open Questions
```

## Pull requests
- Keep PRs small and focused.
- Link related Issue numbers in the PR description.
- Include screenshots or GIFs for UI changes.

## License
No license is selected in the upstream project. If you plan to contribute, please open an Issue to discuss licensing (MIT, Apache-2.0, GPL, or different rules for assets).

Thank you for contributing — every bit helps shape the game!
